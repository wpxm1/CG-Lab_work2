## 项目架构

本项目采用模块化设计，主要包含以下文件：

- `src/Work2/main.py`：项目主入口，实现贝塞尔曲线的绘制和性能优化

## 代码逻辑

1. **初始化**：
   - 使用 Taichi 库初始化 GPU 环境
   - 声明像素缓冲区、GUI 绘制数据缓冲池和曲线坐标 GPU 缓冲区
   - 创建窗口和画布

2. **事件处理**：
   - 响应鼠标左键点击，添加控制点
   - 响应 'c' 键，清空画布和控制点

3. **贝塞尔曲线计算**：
   - 使用 De Casteljau 算法计算曲线点
   - 在 CPU 端计算所有曲线点的坐标
   - 一次性将计算结果发送到 GPU

4. **渲染**：
   - 并行清空像素缓冲区
   - 在 GPU 上并行绘制曲线
   - 绘制控制点和控制线

5. **性能优化**：
   - 使用 GPU 缓冲区存储曲线坐标，减少 CPU-GPU 数据传输
   - 在 GPU 上并行执行像素绘制操作

## 实现功能

- **贝塞尔曲线绘制**：支持任意数量的控制点（最多 100 个）
- **用户交互**：通过鼠标左键添加控制点，按 'c' 键清空画布
- **性能优化**：
  - GPU 加速计算和渲染
  - 减少 CPU-GPU 数据传输
  - 并行处理像素操作
- **可视化**：
  - 绿色曲线：贝塞尔曲线
  - 红色点：控制点
  - 灰色线：控制点连接线

## 技术特点

1. **De Casteljau 算法**：使用递归实现的 De Casteljau 算法计算贝塞尔曲线
2. **GPU 并行计算**：利用 Taichi 的 GPU 后端加速计算和渲染
3. **内存优化**：
   - 预分配 GPU 缓冲区
   - 批量数据传输
   - 避免频繁内存操作
4. **事件驱动**：响应式处理用户输入

## 运行方式

运行后会弹出一个窗口，操作说明：
- **鼠标左键**：点击添加控制点
- **'c' 键**：清空画布和控制点

随着控制点的添加，贝塞尔曲线会实时更新并显示。
